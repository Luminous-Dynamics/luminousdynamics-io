🚀 Getting Started

You can also use our REST API with any language that supports HTTP requests. See our SDK documentation for installation guides.

🔐 Authentication

We use API key authentication. Include your API key in the request headers:

Authorization: Bearer YOUR_API_KEY

Example using curl:

curl -H "Authorization: Bearer lum_sk_..." https://api.luminousdynamics.io/v1/...

Never expose your API keys in client-side code or public repositories. Use environment variables to store them securely.

After signing up (coming soon), you can find your API keys in the Dashboard:

  1. Log in to your account
  2. Navigate to Settings → API Keys
  3. Click "Create New Key"
  4. Copy your key and store it securely

Important: API keys are only shown once. If you lose your key, you'll need to create a new one and rotate your credentials.

Yes! We recommend rotating your API keys regularly for security:

  1. Create a new API key in the dashboard
  2. Update your applications to use the new key
  3. Test that the new key works correctly
  4. Revoke the old key

You can have multiple active keys to facilitate zero-downtime rotation.

💻 API Usage

All our APIs use JSON for both requests and responses:

  • Request: Send JSON in the request body with Content-Type: application/json
  • Response: Receive JSON with proper content type headers
  • Encoding: UTF-8 character encoding

Some endpoints also support URL-encoded form data for simpler requests.

Our APIs use standard HTTP status codes and provide detailed error messages:

  • 200-299: Success
  • 400-499: Client errors (invalid request, authentication failure, etc.)
  • 500-599: Server errors (retry with exponential backoff)

Error responses include a JSON body with error and message fields. See our Best Practices guide for error handling patterns.

Webhook support is coming in Q2 2026! This will include:

  • Real-time event notifications
  • Configurable webhook endpoints
  • Signature verification for security
  • Retry logic for failed deliveries

In the meantime, you can poll our APIs or use WebSocket connections for certain endpoints. Check our Roadmap for updates.

⚡ Rate Limits

Rate limits vary by plan:

  • Free Tier: 100 requests/minute, 10,000 requests/month
  • Pro: 1,000 requests/minute, 1M requests/month
  • Enterprise: Custom limits based on your needs

Rate limit headers are included in all responses to help you track usage. See our Best Practices for handling rate limits gracefully.

Every API response includes rate limit headers:

  • X-RateLimit-Limit - Total requests allowed per window
  • X-RateLimit-Remaining - Requests remaining in current window
  • X-RateLimit-Reset - Unix timestamp when the limit resets

When you exceed the limit, you'll receive a 429 Too Many Requests response with a Retry-After header.

When you exceed your rate limit:

  1. You'll receive a 429 Too Many Requests HTTP status
  2. The response includes a Retry-After header indicating when to retry
  3. Your requests will be rejected until the rate limit window resets

We recommend implementing exponential backoff and respecting the Retry-After header. See our error handling examples for implementation patterns.

🛡️ Security & Privacy

We implement industry-standard security practices:

  • Encryption: TLS 1.3 for all data in transit
  • Storage: AES-256 encryption for data at rest
  • Access: Role-based access control (RBAC)
  • Auditing: Comprehensive audit logs
  • Compliance: SOC 2, GDPR, CCPA compliant

See our Security Policy for details.

We take security seriously. To report a vulnerability:

  1. Email security@luminousdynamics.io
  2. Include detailed steps to reproduce
  3. Use our PGP key for sensitive reports (available on security page)
  4. Allow 90 days for coordinated disclosure

We offer bounties for valid security reports. See our Security Policy for the responsible disclosure process.

API key security best practices:

  • Never commit keys to version control
  • Use environment variables to store keys
  • Rotate keys regularly (every 90 days recommended)
  • Restrict key permissions to minimum required
  • Monitor key usage in your dashboard

If you suspect a key has been compromised, revoke it immediately and create a new one.
