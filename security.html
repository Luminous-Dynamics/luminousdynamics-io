🔒 Security Policy

We take security seriously. This document outlines our security policy and responsible disclosure process.

📢 Reporting a Vulnerability

If you discover a security vulnerability in our platform, we appreciate your help in disclosing it to us responsibly.

Security Contact

📧
Email: security@luminousdynamics.io
🔐
PGP Key: Download PGP Public Key
⏱️
Response Time: Within 48 hours

Please include the following information in your report:

  • Type of vulnerability (e.g., XSS, SQL injection, authentication bypass)
  • Full paths of affected source file(s)
  • Location of the affected code (tag/branch/commit or URL)
  • Step-by-step instructions to reproduce the issue
  • Proof-of-concept or exploit code (if possible)
  • Impact of the vulnerability and how it could be exploited

🔄 Disclosure Process

We follow a coordinated disclosure process to ensure vulnerabilities are handled responsibly:

  • Day 0: You report the vulnerability to our security team
  • Day 1-2: We acknowledge receipt and begin investigation
  • Day 3-14: We work on a fix and coordinate with you on details
  • Day 15-30: We deploy the fix and prepare public disclosure
  • Day 30+: Public disclosure after fix deployment (with your credit, if desired)
⚠️ Important: Please do not publicly disclose the vulnerability until we have had sufficient time to address it. We aim to fix critical issues within 30 days.

🛡️ Our Security Practices

We employ industry-standard security practices to protect our platform and users:

  • All data transmitted over HTTPS with TLS 1.3
  • Regular security audits and penetration testing
  • Dependency scanning and automatic updates
  • Content Security Policy (CSP) headers
  • Subresource Integrity (SRI) for external resources
  • Rate limiting and DDoS protection
  • Secure authentication with modern cryptography
  • Regular security training for development team
