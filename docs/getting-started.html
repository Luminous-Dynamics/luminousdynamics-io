    Getting Started

    Learn how to quickly integrate Luminous Dynamics APIs into your applications. This guide covers authentication, making your first API call, and common patterns.

    📅 Last updated: November 2025 ⏱️ 5 minute read

    Prerequisites

    Before you begin, ensure you have the following:

    • A Luminous Dynamics account (sign up at luminousdynamics.io)
    • An API key from your dashboard
    • Basic knowledge of HTTP/REST APIs
    • A programming language runtime (Node.js, Python, etc.)
    ℹ️ New to APIs?

    If you're new to working with APIs, check out our API Basics Guide for a gentle introduction to HTTP requests, authentication, and JSON.

    Authentication

    All Luminous Dynamics APIs use API key authentication. Your API key should be included in the Authorization header of every request.

    Getting Your API Key

    1. Log in to your dashboard
    2. Navigate to Settings → API Keys
    3. Click Generate New Key
    4. Give your key a descriptive name (e.g., "Production App")
    5. Copy the key immediately (you won't be able to see it again)
    ⚠️ Security Warning

    Never commit API keys to version control. Use environment variables or secure secret management services. Treat API keys like passwords.

    Using Your API Key

    Include your API key in the Authorization header with the Bearer scheme: 

    GET /v1/sites HTTP/1.1
Host: api.luminousdynamics.io
Authorization: Bearer YOUR_API_KEY
Content-Type: application/json

    Environment Variables

    Store your API key in an environment variable: 

    # .env file
LUMINOUS_API_KEY=your_api_key_here
    // Node.js
require('dotenv').config();
const apiKey = process.env.LUMINOUS_API_KEY;
    # Python
import os
from dotenv import load_dotenv

load_dotenv()
api_key = os.getenv('LUMINOUS_API_KEY')

    Your First API Call

    Let's make a simple API call to verify your setup by fetching your account information.

    JavaScript / Node.js

    const apiKey = process.env.LUMINOUS_API_KEY;

async function getAccountInfo() {
  const response = await fetch('https://api.luminousdynamics.io/v1/account', {
    method: 'GET',
    headers: {
      'Authorization': `Bearer ${apiKey}`,
      'Content-Type': 'application/json'
    }
  });

  if (!response.ok) {
    throw new Error(`HTTP ${response.status}: ${response.statusText}`);
  }

  const data = await response.json();
  console.log('Account:', data);
  return data;
}

getAccountInfo();

    Python

    import os
import requests

api_key = os.getenv('LUMINOUS_API_KEY')

def get_account_info():
    url = 'https://api.luminousdynamics.io/v1/account'
    headers = {
        'Authorization': f'Bearer {api_key}',
        'Content-Type': 'application/json'
    }

    response = requests.get(url, headers=headers)
    response.raise_for_status()

    data = response.json()
    print('Account:', data)
    return data

get_account_info()

    cURL

    curl -X GET https://api.luminousdynamics.io/v1/account \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"
    ✓ Success!

    If you received a JSON response with your account details, congratulations! You've successfully authenticated with the Luminous Dynamics API.

    Common Patterns

    Error Handling

    Always implement proper error handling. The API uses standard HTTP status codes:

    • 200 - Success
    • 400 - Bad Request (validation error)
    • 401 - Unauthorized (invalid API key)
    • 404 - Resource not found
    • 429 - Too many requests (rate limit)
    • 500 - Server error
    async function makeRequest(url, options) {
  try {
    const response = await fetch(url, options);

    if (!response.ok) {
      const error = await response.json();
      throw new Error(`${response.status}: ${error.message || response.statusText}`);
    }

    return await response.json();
  } catch (error) {
    console.error('API Error:', error);
    throw error;
  }
}

    Pagination

    List endpoints return paginated results. Use the page and limit parameters: 

    const response = await fetch(
  'https://api.luminousdynamics.io/v1/sites?page=1&limit=20',
  {
    headers: {
      'Authorization': `Bearer ${apiKey}`,
      'Content-Type': 'application/json'
    }
  }
);

const data = await response.json();
console.log(`Total: ${data.total}, Page: ${data.page}/${data.pages}`);

    Filtering and Sorting

    Most endpoints support filtering and sorting via query parameters: 

    // Filter by type and sort by capacity
const url = new URL('https://api.luminousdynamics.io/v1/sites');
url.searchParams.set('type', 'solar');
url.searchParams.set('sort', '-capacity'); // minus for descending
url.searchParams.set('minCapacity', '1000');

const response = await fetch(url, { headers });

    Rate Limiting

    API requests are rate-limited to ensure fair usage. Check response headers: 

    const response = await fetch(url, options);

const rateLimit = {
  limit: response.headers.get('X-RateLimit-Limit'),
  remaining: response.headers.get('X-RateLimit-Remaining'),
  reset: new Date(response.headers.get('X-RateLimit-Reset') * 1000)
};

console.log(`${rateLimit.remaining}/${rateLimit.limit} requests remaining`);
