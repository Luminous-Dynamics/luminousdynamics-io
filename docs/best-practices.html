    Best Practices

    Production-ready patterns and best practices for building reliable, performant, and secure applications with Luminous Dynamics APIs.

    Architecture Patterns

    Client Configuration

    Create a centralized API client instance and reuse it across your application. This ensures efficient connection pooling and consistent configuration. 

    // api/client.js - Centralized client configuration
import { LuminousClient } from '@luminous/api-client';

const client = new LuminousClient({
  apiKey: process.env.LUMINOUS_API_KEY,
  timeout: 10000,
  retries: 3,
  onRateLimit: (retryAfter) => {
    console.warn(`Rate limited. Retrying after ${retryAfter}s`);
  },
  onError: (error) => {
    // Centralized error logging
    console.error('API Error:', error);
  }
});

export default client;

    Service Layer Pattern

    Encapsulate API calls in a service layer to separate concerns and enable easier testing. 

    // services/terra-service.js
import client from '../api/client';

export class TerraService {
  async findSolarSites({ minCapacity, location }) {
    try {
      const sites = await client.terra.sites.list({
        type: 'solar',
        minCapacity,
        ...location
      });

      return this.transformSites(sites);
    } catch (error) {
      throw this.handleError(error);
    }
  }

  transformSites(sites) {
    // Transform API response to match your domain model
    return sites.map(site => ({
      id: site.id,
      displayName: site.name,
      powerOutput: `${site.capacity} kW`,
      coordinates: [site.location.lat, site.location.lng]
    }));
  }

  handleError(error) {
    // Transform API errors to application errors
    if (error.code === 'RATE_LIMIT') {
      return new RateLimitError(error.retryAfter);
    }
    return new ServiceError(error.message);
  }
}

    Error Handling

    Comprehensive Error Handling

    Always implement comprehensive error handling with specific error types. 

    async function callAPI() {
  try {
    const result = await client.nix.analyze({
      data: neuralData,
      depth: 7
    });

    return result;
  } catch (error) {
    // Handle specific error types
    if (error.code === 'UNAUTHORIZED') {
      // Invalid API key - check credentials
      throw new AuthError('Invalid API key. Please check your credentials.');
    }

    if (error.code === 'RATE_LIMIT') {
      // Rate limit exceeded - implement backoff
      const retryAfter = error.retryAfter || 60;
      console.warn(`Rate limited. Retry after ${retryAfter}s`);

      return retryWithBackoff(() => callAPI(), retryAfter);
    }

    if (error.code === 'VALIDATION_ERROR') {
      // Invalid parameters - fix request
      console.error('Validation failed:', error.details);
      throw new ValidationError(error.details);
    }

    if (error.code === 'SERVER_ERROR') {
      // Server error - log and retry
      console.error('Server error:', error);
      return retryWithExponentialBackoff(() => callAPI());
    }

    // Unknown error - log for investigation
    console.error('Unknown API error:', error);
    throw error;
  }
}

    Retry Strategies

    Implement exponential backoff for transient failures. 

    async function retryWithExponentialBackoff(
  fn,
  maxRetries = 3,
  baseDelay = 1000
) {
  for (let attempt = 0; attempt <= maxRetries; attempt++) {
    try {
      return await fn();
    } catch (error) {
      if (attempt === maxRetries) throw error;

      // Exponential backoff: 1s, 2s, 4s, 8s...
      const delay = baseDelay * Math.pow(2, attempt);

      // Add jitter to prevent thundering herd
      const jitter = Math.random() * 1000;

      await sleep(delay + jitter);
    }
  }
}
    ⚠️ Retry Safely

    Only retry idempotent operations (GET, DELETE) automatically. For POST/PUT, implement idempotency keys or verify state before retrying.

    Performance Optimization

    Caching Strategies

    Cache API responses to reduce latency and API usage. 

    import { LRUCache } from 'lru-cache';

const cache = new LRUCache({
  max: 500,
  ttl: 1000 * 60 * 5 // 5 minutes
});

async function getCachedSites(params) {
  const cacheKey = JSON.stringify(params);

  // Check cache first
  const cached = cache.get(cacheKey);
  if (cached) {
    console.log('Cache hit');
    return cached;
  }

  // Fetch from API
  const sites = await client.terra.sites.list(params);

  // Store in cache
  cache.set(cacheKey, sites);

  return sites;
}

    Efficient Pagination

    For large datasets, use cursor-based pagination when available. 

    async function fetchAllSites() {
  const allSites = [];
  let page = 1;
  let hasMore = true;

  while (hasMore) {
    const response = await client.terra.sites.list({
      page,
      limit: 100 // Fetch in batches
    });

    allSites.push(...response.data);

    hasMore = page < response.pages;
    page++;

    // Avoid hitting rate limits
    if (hasMore) {
      await sleep(100); // Small delay between pages
    }
  }

  return allSites;
}

    Parallel Requests

    Execute independent API calls in parallel to reduce total latency. 

    // ❌ Sequential - slow
const solarSites = await client.terra.sites.list({ type: 'solar' });
const windSites = await client.terra.sites.list({ type: 'wind' });
const hydroSites = await client.terra.sites.list({ type: 'hydro' });

// ✅ Parallel - fast
const [solarSites, windSites, hydroSites] = await Promise.all([
  client.terra.sites.list({ type: 'solar' }),
  client.terra.sites.list({ type: 'wind' }),
  client.terra.sites.list({ type: 'hydro' })
]);

    Security Best Practices

    API Key Security

    Never expose API keys in client-side code or version control.

    🔒 Critical
    NEVER do this:
    • Hardcode API keys in source code
    • Commit API keys to git
    • Use API keys in client-side JavaScript
    • Share API keys in public channels

    Backend Proxy Pattern

    For client-side applications, proxy API calls through your backend. 

    // Backend API route
app.post('/api/analyze', async (req, res) => {
  // Validate user authentication
  if (!req.user) {
    return res.status(401).json({ error: 'Unauthorized' });
  }

  // Validate request
  if (!req.body.data) {
    return res.status(400).json({ error: 'Missing data' });
  }

  try {
    // Call Luminous API with server-side key
    const result = await client.nix.analyze({
      data: req.body.data,
      depth: req.body.depth || 5
    });

    res.json(result);
  } catch (error) {
    console.error('API error:', error);
    res.status(500).json({ error: 'Analysis failed' });
  }
});

    Rate Limit Management

    Track Rate Limits

    Monitor rate limit headers to prevent hitting limits. 

    class RateLimitTracker {
  constructor() {
    this.remaining = null;
    this.limit = null;
    this.reset = null;
  }

  update(response) {
    this.remaining = parseInt(response.headers.get('X-RateLimit-Remaining'));
    this.limit = parseInt(response.headers.get('X-RateLimit-Limit'));
    this.reset = new Date(response.headers.get('X-RateLimit-Reset') * 1000);
  }

  shouldThrottle() {
    // Throttle when under 10% remaining
    return this.remaining < (this.limit * 0.1);
  }

  getWaitTime() {
    if (!this.reset) return 0;
    return Math.max(0, this.reset - Date.now());
  }
}

const rateLimiter = new RateLimitTracker();

    Testing

    Mock API Calls

    Use mocks for unit testing to avoid hitting the API. 

    // __tests__/terra-service.test.js
import { TerraService } from '../services/terra-service';
import client from '../api/client';

jest.mock('../api/client');

describe('TerraService', () => {
  beforeEach(() => {
    jest.clearAllMocks();
  });

  it('finds solar sites', async () => {
    const mockSites = [
      { id: '1', name: 'Solar Farm', capacity: 5000 }
    ];

    client.terra.sites.list.mockResolvedValue(mockSites);

    const service = new TerraService();
    const sites = await service.findSolarSites({ minCapacity: 1000 });

    expect(client.terra.sites.list).toHaveBeenCalledWith({
      type: 'solar',
      minCapacity: 1000
    });
    expect(sites).toHaveLength(1);
  });
});

    Use a test API key for integration tests.

    Monitoring

    Implement comprehensive monitoring and alerting. 

    import { logger } from './logger';
import { metrics } from './metrics';

const monitoredClient = new LuminousClient({
  apiKey: process.env.LUMINOUS_API_KEY,

  onRequest: (config) => {
    metrics.increment('api.requests', {
      endpoint: config.url,
      method: config.method
    });

    logger.debug('API Request', {
      url: config.url,
      method: config.method
    });
  },

  onResponse: (response) => {
    metrics.timing('api.response_time', response.duration);
    metrics.increment('api.success', {
      endpoint: response.url,
      status: response.status
    });
  },

  onError: (error) => {
    metrics.increment('api.errors', {
      code: error.code,
      endpoint: error.url
    });

    logger.error('API Error', {
      code: error.code,
      message: error.message,
      url: error.url
    });

    // Alert on critical errors
    if (error.code === 'UNAUTHORIZED') {
      alerts.critical('Invalid API key detected');
    }
  }
});
